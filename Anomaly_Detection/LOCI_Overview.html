LOCI (Local Correlation Integral) Overview Guide

Density-based anomaly detection: identify outliers using multi-granularity deviation factor (MDEF) — evaluates points across multiple neighborhood sizes to detect anomalies.

Primary Uses

Anomaly detection, local outlier identification, fraud detection, network intrusion detection, detecting isolated points across multiple scales, identifying outliers in data with varying densities and scales.

Strengths

Multi-granularity approach (no single k parameter), automatically adapts to different scales, detects outliers at multiple resolutions, no fixed neighborhood size required, provides interpretable MDEF scores, handles varying density regions effectively.

Watch-outs

Computationally expensive O(n²) for naive implementation, sensitive to rmax parameter selection, requires normalization of features, threshold (n-sigma) selection is critical, can be affected by curse of dimensionality, more complex than LOF.

How It Works Mathematical Foundation

The LOCI (Local Correlation Integral) method evaluates points across multiple neighborhood sizes (r-neighborhoods) from small to large (up to rmax). It uses the MDEF (Multi-granularity Deviation Factor) score: MDEF(r) = (n_p(r) - n_avg(r)) / n_avg(r), where n_p(r) is the number of neighbors within radius r and n_avg(r) is the average neighbor count across all points.

  • rmax Parameter: maximum radius to consider (typically based on data scale)
  • alpha Parameter: granularity control (typically 0.5, determines r values to test)
  • n_p(r): number of neighbors within radius r from point p
  • n_avg(r): average neighbor count across all points at radius r
  • MDEF Score: high MDEF indicates significantly fewer neighbors than average (anomaly)
  • n-sigma Threshold: typically 3 standard deviations; points with MDEF > n-sigma are anomalies
Tip: MDEF interpretation: MDEF ≈ 0 means normal (similar neighbor count to average), MDEF > 0.5 suggests potential outlier, MDEF > 1.0 indicates strong outlier. The n-sigma threshold (typically 3) provides statistical significance.

  • MDEF Score Heatmap: color-coded MDEF values across the space; brighter colors = higher MDEF (more anomalous).

  • MDEF Distribution: histogram helps set threshold; 3D typically shows different MDEF patterns.
  • Anomaly Points: clearly marked in red; inspect from different angles to understand multi-scale context.

Parameter Guidance Best Practice

  • rmax Parameter: Set based on data scale. Too small → misses large-scale outliers. Too large → computationally expensive. Start with rmax = 2-3 × average inter-point distance. For 2D, typically 0.5-2.0. For 3D, typically 1.0-3.0.
  • alpha Parameter: Controls granularity of r values tested. alpha = 0.5 is standard (tests r = alpha × rmax, alpha² × rmax, ...). Smaller alpha → more granular but slower. Typically 0.3-0.7.
  • n-sigma Threshold: Statistical threshold (standard deviations). n-sigma = 3 is standard (99.7% confidence). Lower (2) → more sensitive, higher (4) → more conservative. Typically 2-4.
  • Normalization: Essential! Normalize features to same scale, otherwise features with larger scales dominate the distance calculation.

Data Preparation Best Practice

  • Normalize all features to the same scale (e.g., min-max normalization, z-score standardization).
  • Handle missing values before computing distances (impute or remove).
  • For high-dimensional data, consider dimensionality reduction (PCA) to avoid curse of dimensionality.
  • Ensure sufficient data points; rmax should be appropriate for data scale.
  • Remove or handle extreme outliers in training data if they skew the average neighbor count calculations.

Common Pitfalls Avoid

  • Too small rmax → misses large-scale outliers, only detects local anomalies.
  • Too large rmax → computationally expensive, may include irrelevant distant points.
  • Not normalizing features → features with larger scales dominate distance calculation.
  • Too low n-sigma threshold → too many false positives (normal points flagged as anomalies).
  • Too high n-sigma threshold → misses real anomalies (anomalies not detected).
  • Curse of dimensionality → in high dimensions, all distances become similar; consider dimensionality reduction.
  • Computational cost → naive implementation is O(n² × number of r values); use optimized data structures for large datasets.

LOCI vs LOF Comparison

  • Multi-granularity vs Single k: LOCI evaluates multiple neighborhood sizes; LOF uses single k parameter.
  • Adaptability: LOCI automatically adapts to different scales; LOF requires choosing appropriate k.
  • Computation: LOCI is more expensive (multiple r values); LOF is faster but less adaptive.
  • Parameter Selection: LOCI requires rmax and alpha; LOF requires k parameter.
  • Use Case: LOCI for multi-scale outliers, varying density regions; LOF for single-scale local outliers.
  • Score Interpretation: LOCI uses MDEF (relative deviation); LOF uses density ratio.

When to Use Application

  • Multi-scale Anomaly Detection: identify outliers that appear at different scales or resolutions.
  • Fraud Detection: detect unusual transaction patterns that are isolated across multiple scales.
  • Network Security: detect unusual network traffic patterns at various time scales.
  • Quality Control: detect manufacturing defects or isolated measurements across different scales.
  • Varying Density Data: detect outliers in data with clusters of varying densities and sizes.
  • Scale-agnostic Detection: when you don't know the appropriate neighborhood size in advance.
