Variance Threshold Overview Guide

A simple filter method that removes features with variance below a threshold — fast, unsupervised feature selection.

Primary Uses

Preprocessing, removing constant/near-constant features, dimensionality reduction, data cleaning, feature selection, removing low-information features.

Strengths

Extremely fast (O(n×p)), simple to understand and implement, no hyperparameters (just threshold), unsupervised (no labels needed), removes truly uninformative features, interpretable results.

Watch-outs

Assumes low variance = low information (not always true), sensitive to feature scaling, may remove important features with small but meaningful variance, doesn't consider feature relationships or target variable.

How It Works Mathematical Foundation

Variance Threshold is a filter method that removes features based on their variance. The assumption is that features with very low variance (near-constant values) provide little information and can be safely removed. It computes the variance of each feature and keeps only those with variance above a specified threshold.

  • Variance Calculation: For each feature j, compute Var(Xⱼ) = (1/(n-1)) Σᵢ (xᵢⱼ - μⱼ)² where μⱼ is the mean of feature j
  • Threshold Comparison: Compare each feature's variance to the threshold value
  • Feature Selection: Keep features where Var(Xⱼ) ≥ threshold, remove features where Var(Xⱼ) < threshold
  • Zero Variance: Features with zero variance (constant values) are always removed regardless of threshold
  • Scaling Sensitivity: Variance is scale-dependent, so features with larger scales will have higher variance
  • Binary Features: For binary features, variance = p(1-p) where p is the proportion of 1s. Maximum variance (0.25) occurs when p=0.5
Tip: Always consider feature scaling before applying Variance Threshold. Features with different scales will have incomparable variances. Standardize features (mean=0, std=1) if you want to compare variances across features on equal footing. However, if features are already on similar scales, you can apply Variance Threshold directly.

Reading the Visualizations Analysis

  • Variance Distribution: Histogram showing variance of all features. Helps identify natural breakpoints and appropriate threshold values. Features with very low variance appear as a spike near zero.
  • Variance vs Threshold: Shows which features are kept (above threshold line) and removed (below threshold line). Adjust threshold to see impact on feature selection.
  • Feature Comparison: Before/after comparison showing which features were removed. Useful for understanding what information was discarded.
  • Variance Heatmap: Visual representation of feature variances, sorted by variance. Makes it easy to identify low-variance features.
  • Threshold Sensitivity: Shows how the number of selected features changes with different threshold values. Helps choose optimal threshold.

Parameter Guidance Best Practice

  • Threshold Value: Common values: 0.0 (removes only zero-variance features), 0.01-0.1 (removes near-constant features), or use variance distribution to find natural breakpoint. For binary features, max variance is 0.25.
  • Feature Scaling: Consider standardizing features before applying threshold if features have different scales. This ensures fair comparison of variances across features.
  • Zero Variance Features: Always remove features with zero variance (constant values) as they provide no information. Set threshold = 0.0 to remove only these.
  • Binary Features: For binary features, variance = p(1-p). If a binary feature has variance < 0.01, it means p is very close to 0 or 1 (highly imbalanced), which may be uninformative.
  • Domain Knowledge: Consider domain-specific variance expectations. Some features may naturally have low variance but still be important (e.g., rare events).

Data Preparation Best Practice

  • Handle Missing Values: Remove or impute missing values before computing variance. Variance calculation requires complete data.
  • Feature Scaling (Optional): If features have different scales, consider standardizing (mean=0, std=1) so variances are comparable. However, if you want to preserve original feature scales, apply threshold directly.
  • Remove Constant Features: Features with zero variance (all values identical) should always be removed. They provide no information.
  • Numeric Features Only: Variance Threshold works only on numeric features. Encode categorical variables appropriately first (e.g., one-hot encoding).
  • Sample Size: Variance estimates become more stable with larger sample sizes. For small datasets, be cautious with very low thresholds.

Common Pitfalls Avoid

  • Ignoring Feature Scaling: Features with larger scales will have higher variance, leading to biased selection. Standardize if features have different scales.
  • Too High Threshold: Setting threshold too high may remove important features with meaningful but small variance (e.g., rare but important events).
  • Too Low Threshold: Setting threshold too low (e.g., 0.0) only removes zero-variance features, missing near-constant features that add noise.
  • Assuming Low Variance = Unimportant: Some important features may have low variance (e.g., rare disease indicators). Consider domain knowledge.
  • Not Checking Removed Features: Always inspect which features were removed to ensure no important information was discarded.
  • Applying to Categorical Data: Variance Threshold requires numeric data. Encode categorical variables first (e.g., one-hot encoding for binary features).
  • Ignoring Feature Relationships: Variance Threshold doesn't consider feature correlations or relationships with target variable. Use in combination with other methods.

When to Use Application

  • Preprocessing Step: As a first step in feature selection pipeline to remove obviously uninformative features before applying more sophisticated methods.
  • High-Dimensional Data: When you have many features (p > 100) and want to quickly remove low-variance features to reduce dimensionality.
  • Constant Feature Removal: When you want to automatically identify and remove features with zero or near-zero variance.
  • Data Cleaning: When cleaning datasets that may contain constant or near-constant features (e.g., from data collection errors).
  • Fast Feature Selection: When you need a very fast feature selection method for large datasets (O(n×p) complexity).
  • Baseline Method: As a simple baseline before applying more sophisticated feature selection methods (wrapper or embedded methods).
  • Binary Feature Filtering: When working with binary features and want to remove highly imbalanced features (variance < 0.01 indicates p close to 0 or 1).

Alternatives to Consider Comparison

  • vs Wrapper Methods (e.g., RFE): Variance Threshold is fast but doesn't consider feature interactions or target variable. Wrapper methods are slower but more accurate. Use Variance Threshold for preprocessing, wrapper methods for final selection.
  • vs Embedded Methods (e.g., Lasso): Variance Threshold is unsupervised and simple. Embedded methods consider target variable and feature interactions but require model training. Use Variance Threshold for quick filtering, embedded methods for model-aware selection.
  • vs Correlation-Based Filters: Variance Threshold removes low-variance features. Correlation filters remove highly correlated features. Use both: first remove low variance, then remove high correlation.
  • vs Univariate Statistical Tests: Variance Threshold is unsupervised. Statistical tests (e.g., chi-square, ANOVA) consider target variable. Use Variance Threshold first, then statistical tests for supervised selection.
  • vs PCA: Variance Threshold removes features. PCA creates new features (components). Use Variance Threshold to reduce feature count, PCA to reduce dimensionality while preserving variance.
