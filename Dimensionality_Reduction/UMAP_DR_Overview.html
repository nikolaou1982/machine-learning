UMAP Overview Guide

Uniform Manifold Approximation and Projection: fast, scalable dimensionality reduction that preserves both local and global structure using Riemannian geometry and fuzzy simplicial sets.

Primary Uses

Data visualization, dimensionality reduction, preserving global structure, fast computation on large datasets, single-cell RNA-seq analysis, exploratory data analysis, feature extraction, can transform new data points.

Strengths

Faster than t-SNE, preserves global structure better, can transform new data points, scalable to large datasets, deterministic results (with fixed random seed), handles non-linear manifolds, preserves both local and global relationships.

Watch-outs

Requires tuning of n_neighbors and min_dist parameters, computationally expensive for very large datasets, memory intensive, sensitive to parameter choices, may require preprocessing for best results.

How It Works Mathematical Foundation

UMAP constructs a high-dimensional weighted graph representing the data manifold, then finds a low-dimensional representation that preserves the topological structure of this graph. It uses fuzzy simplicial sets and Riemannian geometry to model the manifold structure.

  • High-Dimensional Graph: For each point, find k nearest neighbors and compute distances. Create a weighted graph where edge weights represent local manifold structure.
  • Fuzzy Simplicial Sets: Convert distances to probabilities using a local metric: ρi = distance to nearest neighbor, σi chosen so that Σj exp(-max(0, d(xi, xj) - ρi) / σi) = log2(k).
  • High-Dimensional Probabilities: pj|i = exp(-max(0, d(xi, xj) - ρi) / σi), then symmetrize: pij = pj|i + pi|j - pj|ipi|j.
  • Low-Dimensional Probabilities: Use a smooth approximation: qij = (1 + a(||yi - yj||2b))-1, where a and b are chosen based on min_dist.
  • Cross-Entropy Loss: Minimize C = Σij [pij log(pij/qij) + (1 - pij) log((1 - pij)/(1 - qij))].
  • Gradient Descent: Use stochastic gradient descent with negative sampling for efficiency.
  • Riemannian Geometry: Assumes data lies on a Riemannian manifold and preserves the manifold's topological structure.
Tip: n_neighbors balances local vs global structure (typically 15-50). Lower values emphasize local structure; higher values preserve more global structure. min_dist controls how tightly points are packed in the embedding (typically 0.1-0.5). Lower values = tighter clusters, higher values = more spread out.

Reading the Visualizations Analysis

  • 2D Embedding Plot: Visualize clusters and structure; should show both local neighborhoods and global relationships.
  • Cross-Entropy Loss History: Monitor convergence; should decrease and stabilize. Faster convergence than t-SNE.
  • Probability Comparison: Compare high-dimensional vs low-dimensional probabilities; good preservation = points align along diagonal.
  • Global Structure: Unlike t-SNE, UMAP preserves global structure—distant points in high-D should be distant in low-D.
  • Cluster Separation: Well-separated clusters indicate good local structure preservation.

Parameter Guidance Best Practice

  • n_neighbors: Typically 15-50. Lower = more local structure, higher = more global structure. Start with 15-30.
  • min_dist: Typically 0.1-0.5. Controls minimum distance between points in embedding. Lower = tighter clusters, higher = more spread out. Start with 0.1.
  • n_components: Usually 2-3 for visualization. Higher dimensions preserve more structure but harder to visualize.
  • metric: Distance metric (Euclidean, Manhattan, etc.). Euclidean is default and works well for most cases.
  • Normalization: Standardize features before UMAP to treat all dimensions uniformly.
  • Random Seed: Set for reproducibility (UMAP is deterministic with fixed seed).

Data Preparation Checklist

  • Normalize or standardize features (critical for distance-based methods).
  • Ensure sufficient sample size—UMAP works well with hundreds to millions of samples.
  • Impute or remove missing values—distance computation requires complete data.
  • Consider dimensionality reduction (e.g., PCA) before UMAP for very high-dimensional data to reduce noise.
  • Remove outliers that could distort the manifold structure.

Common Pitfalls Avoid

  • n_neighbors too low: Emphasizes very local structure; embedding becomes fragmented, loses global relationships.
  • n_neighbors too high: Considers too many neighbors; loses local structure, approaches PCA-like behavior.
  • min_dist too low: Points pack too tightly; clusters become indistinguishable.
  • min_dist too high: Points spread too far; loses local structure, embedding becomes too sparse.
  • Not normalizing: Features with different scales dominate the distance metric.
  • Ignoring global structure: Unlike t-SNE, UMAP preserves global structure—use this advantage.

When to Use Application

  • Visualizing high-dimensional data when you need both local and global structure preservation.
  • Large-scale dimensionality reduction (faster than t-SNE).
  • When you need to transform new data points (unlike t-SNE).
  • Single-cell RNA-seq and other high-dimensional biological data visualization.
  • Exploratory data analysis on large datasets.
  • When global structure is important (e.g., preserving distances between clusters).

Alternatives to Consider Comparison

  • t-SNE: Better for local structure visualization, but slower and doesn't preserve global structure or transform new points.
  • PCA: Much faster, preserves global structure, but only linear relationships.
  • Isomap: Preserves geodesic distances; better for manifold unfolding, but slower.
  • Autoencoders: Can transform new points, but requires training and may be slower.
  • TriMap: Similar to UMAP but uses triplets instead of pairs, may preserve more global structure.
