How It Works Mathematical Foundation

Mapper Algorithm is a topological data analysis method that creates a graph representation of data by combining filter functions, covering, and clustering. It provides a multi-scale view of data structure through a network of clusters.

  • Filter Function: Maps data points to a reference space (usually 1D or 2D). Common filters: PCA projection, distance to a point, density, or custom functions. The filter determines which aspects of the data to emphasize.
  • Covering: Partitions the filter space into overlapping intervals/bins. Number of intervals and overlap percentage control the resolution and connectivity of the resulting graph. Overlap ensures clusters from adjacent bins can connect.
  • Inverse Image: For each bin, collects all data points whose filter values fall within that bin. Points can belong to multiple bins if bins overlap.
  • Clustering: Within each bin, clusters the points using a clustering algorithm (e.g., single linkage, DBSCAN, k-means). Each cluster becomes a node in the final graph.
  • Graph Construction: Connects clusters (nodes) that share at least one data point. Edges represent overlap between clusters from different bins. The resulting graph captures the topological structure of the data.
  • Node Attributes: Each node represents a cluster and can be colored/sized by cluster size, average filter value, or other statistics. Nodes reveal local structure.
  • Edge Attributes: Edges connect overlapping clusters. Edge weights can represent the number of shared points or other similarity measures.
Tip: The Mapper algorithm is particularly useful for visualizing high-dimensional data structure. Choose a filter function that captures important aspects of your data (e.g., first PCA component for linear structure, density for clusters). Use moderate overlap (20-50%) to ensure connectivity while avoiding too many edges. The resulting graph reveals how clusters relate to each other.

Reading the Visualizations Analysis

  • Mapper Graph: Network visualization where nodes represent clusters and edges connect overlapping clusters. Large nodes = large clusters. Dense regions = many overlapping clusters. Isolated nodes = unique clusters.
  • Filter Space Visualization: Shows how data points map to the filter space (e.g., 1D line or 2D plane). Bins are shown as intervals/rectangles. Points colored by their cluster membership.
  • Original Data Projection: Original data points colored by their cluster/node membership. Shows which points belong to which clusters in the Mapper graph.
  • Node Statistics: Size, color, or other attributes of nodes reveal cluster properties. Large nodes = many points. Color can represent average filter value or other statistics.
  • Edge Statistics: Edge thickness or color can represent the number of shared points or similarity between clusters. Thick edges = strong overlap.
  • Graph Layout: Force-directed or other layouts position nodes to reveal structure. Closely positioned nodes = similar clusters. Branches = different data regions.

Parameter Guidance Best Practice

  • Filter Function: Choose based on data structure. PCA = linear structure, density = clusters, distance = proximity. Can use 1D (single component) or 2D (two components) filters.
  • Number of Intervals: Controls resolution. More intervals = finer structure but more nodes. Typically 5-20 intervals. Too many = fragmented graph. Too few = coarse structure.
  • Overlap Percentage: Controls connectivity. Higher overlap = more edges, more connected graph. Typically 20-50%. Too low = disconnected graph. Too high = too many edges, cluttered graph.
  • Clustering Method: Single linkage = connected components, DBSCAN = density-based, k-means = partition-based. Choice affects cluster shape and graph structure.
  • Clustering Parameters: For DBSCAN: min_samples and eps. For k-means: number of clusters. For single linkage: distance threshold. Adjust based on data density and desired granularity.
  • Distance Metric: Euclidean is standard. Can use Manhattan, cosine, or custom metrics. Affects clustering and graph structure.
  • Normalization: Normalize features before applying filter and clustering. Critical for meaningful results, especially with mixed-scale features.

Data Preparation Best Practice

  • Normalization: Normalize features to [0,1] or standardize. Critical for filter functions and clustering. Use MinMaxScaler or StandardScaler.
  • Handle Missing Values: Remove or impute missing values. Mapper requires complete data for filter functions and clustering.
  • Feature Scaling: Ensure all features are on similar scales. Features with larger ranges will dominate filter functions and distance calculations.
  • Sample Size: Works well with moderate to large datasets (n = 50-1000). Very large datasets (n > 5000) may require sampling or efficient clustering methods.
  • Dimensionality: Works with high-dimensional data. Filter function reduces dimensionality for visualization. Original data can have many features.
  • Filter Function Choice: Choose filter function based on data characteristics. PCA for linear structure, density for clusters, custom functions for domain-specific features.

Common Pitfalls Avoid

  • Not Normalizing Data: Features on different scales cause filter functions and clustering to be dominated by large-scale features. Always normalize!
  • Too Many Intervals: Excessive intervals create too many small clusters, fragmenting the graph and making it hard to interpret. Use moderate number (5-20).
  • Too Little Overlap: Insufficient overlap creates disconnected graph components, missing important relationships. Use 20-50% overlap.
  • Wrong Filter Function: Choosing inappropriate filter function misses important structure. Consider multiple filter functions and compare results.
  • Inappropriate Clustering: Clustering method or parameters that don't match data characteristics produce poor clusters and misleading graph structure.
  • Ignoring Graph Structure: Focus only on nodes, ignoring edges and overall graph topology misses important relationships and structure.
  • Over-interpretation: Not all graph features are meaningful. Consider data context and domain knowledge when interpreting results.

When to Use Application

  • High-Dimensional Data Visualization: When you want to visualize structure in high-dimensional data through a graph representation.
  • Structure Discovery: When you want to discover hidden structure, relationships, and patterns in complex datasets.
  • Clustering Visualization: When you want to visualize how clusters relate to each other and understand cluster relationships.
  • Network Analysis: When you want to analyze data as a network/graph to understand connectivity and topology.
  • Biological Data: When analyzing biological data (gene expression, protein structures, neural networks) where graph structure is meaningful.
  • Time Series Analysis: When analyzing time series data to detect patterns, cycles, and relationships through graph structure.
  • Feature Extraction: When you want to extract graph-based features for downstream analysis or machine learning.

Alternatives to Consider Comparison

  • vs Clustering: Mapper provides graph structure showing how clusters relate, while clustering only provides cluster assignments without relationships.
  • vs Persistent Homology: Persistent Homology tracks topological features across scales. Mapper creates a single graph representation with filter-based structure.
  • vs t-SNE/UMAP: These methods create 2D embeddings. Mapper creates a graph that can reveal more complex structure and relationships.
  • vs Graph Neural Networks: GNNs learn from graph structure. Mapper creates graph structure from data for analysis and visualization.
  • vs Network Analysis: Network methods analyze existing graphs. Mapper constructs graphs from data using filter functions and clustering.
  • vs Hierarchical Clustering: Hierarchical clustering creates tree structure. Mapper creates general graph structure with cycles and branches.
