K-means Overview Guide

Centroid-based clustering: partition data into k spherical clusters by minimizing within-cluster variance.

Primary Uses

Customer segmentation, image compression, market research, document clustering, color quantization, vector quantization.

Strengths

Fast and scalable, works well with spherical clusters, simple interpretation, good for large datasets.

Watch-outs

Assumes spherical clusters, requires knowing k, sensitive to initialization, struggles with varying cluster sizes/densities.

How It Works Short

Initialize k centroids randomly. Iterate: assign points to nearest centroid, update centroids to cluster means. Converges when assignments stabilize.

  • k: number of clusters (use elbow method or domain knowledge)
  • Max Iterations: stop after N iterations even if not converged (prevents infinite loops)
  • Initialization: k-means++ improves convergence vs random (used by default)
Tip: Run multiple times with different seeds and pick the best result (lowest inertia/sse).

Reading the Graphs 2D

  • Original Data: raw geometry; identify potential clusters and their count.
  • Clustering Results: color-coded assignments; centroids shown as larger markers; validate separation.
  • Inertia vs k: elbow plot to find optimal k (look for diminishing returns).
  • Silhouette vs k: peaks indicate better k values; negative scores suggest poor fit.
  • Iteration Convergence: inertia decreases per iteration; flat line = converged.

Reading the Graphs 3D

  • Original 3D: spatial structure; rotate to inspect cluster shapes and overlap.
  • Clustering 3D: depth reveals layering; centroids shown as larger 3D markers.
  • Inertia vs k: same elbow method; expect higher inertia values vs 2D for similar scales.
  • Silhouette vs k: use peaks to validate k choices; 3D often needs more clusters than 2D.
  • Iteration Convergence: monitor convergence rate; may need more iterations in 3D.

Parameter Guidance Best Practice

  • k: Start with domain knowledge or use elbow method. Try k = sqrt(n/2) as heuristic.
  • Max Iterations: Default 300 usually sufficient; increase if convergence fails.
  • Multiple Runs: Run 10+ times and select lowest inertia to avoid local minima.
  • Initialization: k-means++ (default) better than random; k-means|| for very large datasets.

Data Preparation Best Practice

  • Standardize or normalize features; remove outliers that distort centroids.
  • Works best with spherical clusters; consider spectral clustering or GMM for non-spherical shapes.
  • For varying sizes/densities, consider k-medoids or hierarchical methods.

Common Pitfalls Avoid

  • Choosing k too high → overfitting, too low → underfitting (use validation metrics).
  • Poor initialization → local minima (run multiple times, use k-means++).
  • Non-spherical data → poor results (try alternative algorithms).
  • Not standardizing → features with larger scales dominate distance calculations.
