K-Medoids (PAM) Overview Guide

Partitioning Around Medoids: robust centroid-based clustering using actual data points as cluster centers.

Primary Uses

Outlier-robust clustering, customer segmentation with noisy data, image compression with robustness, market research, any scenario where k-means fails due to outliers.

Strengths

Robust to outliers, uses actual data points (interpretable medoids), works with any distance metric, more stable than k-means on noisy data.

Watch-outs

Slower than k-means (O(n²) vs O(nk)), computationally expensive for large datasets, still assumes k is known, may struggle with very large k values.

How It Works Short

Select k medoids (actual data points). Iterate: assign points to nearest medoid, swap medoids to reduce total distance. PAM algorithm finds optimal medoids through greedy swaps.

  • k: number of clusters (use elbow method or domain knowledge)
  • Max Iterations: stop after N iterations even if not converged
  • Initialization: BUILD phase selects initial medoids via distance-based heuristics
  • PAM: SWAP phase iteratively improves medoids to minimize total dissimilarity
Tip: K-Medoids is preferable over k-means when outliers are present, as medoids are less affected by extreme values than centroids.

Reading the Graphs 2D

  • Original Data: raw geometry; identify potential clusters and their count; note any outliers.
  • Clustering Results: color-coded assignments; medoids shown as larger markers (actual data points); validate separation and robustness.
  • Total Cost vs k: elbow plot to find optimal k (look for diminishing returns); compare with k-means.
  • Silhouette vs k: peaks indicate better k values; typically more stable than k-means on noisy data.
  • Iteration Convergence: total cost decreases per iteration; flat line = converged (may take more iterations than k-means).

Reading the Graphs 3D

  • Original 3D: spatial structure; rotate to inspect cluster shapes and overlap; identify outliers.
  • Clustering 3D: depth reveals layering; medoids shown as larger 3D markers (actual data points).
  • Total Cost vs k: same elbow method; expect higher cost values vs 2D for similar scales.
  • Silhouette vs k: use peaks to validate k choices; 3D often needs more clusters than 2D.
  • Iteration Convergence: monitor convergence rate; PAM may require more iterations than k-means due to swap complexity.

Parameter Guidance Best Practice

  • k: Start with domain knowledge or use elbow method. Similar to k-means, try k = sqrt(n/2) as heuristic.
  • Max Iterations: Default 300 usually sufficient; PAM may converge slower than k-means, so increase if needed.
  • Initialization: BUILD phase uses distance-based selection; multiple random starts can help avoid poor local optima.
  • When to Use: Prefer over k-means when outliers matter, data is noisy, or you need interpretable cluster centers (actual points).

Data Preparation Best Practice

  • Standardize or normalize features for meaningful distances.
  • K-Medoids is more robust to outliers than k-means, but extreme outliers can still affect results.
  • Works best with spherical clusters; consider spectral clustering or GMM for non-spherical shapes.
  • For very large datasets, consider CLARA (Clustering Large Applications) variant of PAM.

K-Medoids vs K-means Comparison

  • Robustness: Medoids are actual data points, making PAM more robust to outliers than k-means centroids.
  • Speed: K-means is faster (O(nk) per iteration) vs PAM (O(n²) for swaps).
  • Cost Function: PAM minimizes sum of distances to medoids; k-means minimizes squared distances to centroids.
  • Initialization: PAM BUILD phase is deterministic; k-means++ is probabilistic but faster.
  • Interpretability: Medoids are real data points (interpretable); centroids may not correspond to actual data.

Common Pitfalls Avoid

  • Choosing k too high → overfitting, too low → underfitting (use validation metrics).
  • Using on very large datasets → slow convergence (consider CLARA variant or subsampling).
  • Not standardizing → features with larger scales dominate distance calculations.
  • Expecting spherical clusters → PAM still favors compact, well-separated clusters.
