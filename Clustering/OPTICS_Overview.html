OPTICS Overview Guide

Ordering Points To Identify Clustering Structure: density-ordered reachability visualization for variable-density clustering.

Primary Uses

Variable-density datasets, hierarchical structure exploration, reachability plot analysis, finding density valleys/peaks.

Strengths

No single eps needed; reachability reveals structure; handles varying densities; ordering shows natural clusters.

Watch-outs

Threshold selection for cluster extraction; interpretation of reachability valleys; computational cost on large datasets.

How It Works Short

Orders points by density-reachability, building a reachability plot where valleys = clusters and peaks = noise. Cluster extraction uses a threshold on reachability distances.

  • min_samples: core point definition (typical: 2D≈5–10, 3D≈5–15)
  • max_eps: search limit (Infinity for full ordering; smaller for efficiency)
  • threshold: extraction cutoff (adjust interactively via reachability plot)
Tip: Start with a low threshold and increase until clusters separate clearly in reachability valleys.

Validation Metrics Important

Silhouette Score — Use with Caution for Relative Comparison:
  • Silhouette assumes compact, well-separated clusters; OPTICS finds arbitrary shapes and may have overlapping density regions.
  • The score depends on the chosen threshold — different thresholds yield different clusterings, so scores are not directly comparable across threshold values.
  • Low silhouette can indicate density bridges (expected in OPTICS) rather than poor clustering; rely on the reachability plot for validation.
  • Best practice: Compare silhouette only within the same threshold when tuning other parameters (min_samples, max_eps). For cluster selection, prioritize the reachability plot valleys over silhouette alone.
DBCV Score (Density-Based Cluster Validation):
  • Higher is better: DBCV evaluates clusters based on density separation (how well-separated clusters are) and density connectivity (how well-connected points within clusters are).
  • More appropriate for density-based algorithms like OPTICS than silhouette alone; accounts for density structure rather than assuming compact shapes.
  • Use DBCV alongside reachability plot valleys to validate cluster quality; higher DBCV with clear valleys indicates good clustering.
  • Range approximately -1 to 1; values near 1 indicate well-separated, dense clusters; negative values suggest poor cluster structure.

Reading the Graphs 2D

  • Original Data: raw geometry; note density variations and potential clusters.
  • Reachability Plot: valleys = clusters, peaks = noise/gaps; adjust threshold slider to extract clusters at different levels.
  • Core Distance: density measure per point; undefined (Infinity) for noise points.
  • OPTICS Ordering: traversal path showing density-ordered structure.
  • Cluster Labels: extracted clusters at current threshold; adjust threshold to explore different granularities.

Reading the Graphs 3D

  • Original 3D Data: spatial structure; rotate to inspect overlap and separation.
  • Reachability Plot: same interpretation as 2D; valleys indicate clusters in 3D space.
  • Core Distance: 3D density measure; helps identify core vs noise in spatial context.
  • 3D OPTICS Ordering Path: 3D traversal showing how ordering follows density in space.
  • 3D Cluster Labels: spatial clusters at threshold; adjust threshold to reveal multi-scale structure.

Data Preparation Best Practice

  • Standardize/normalize features; remove duplicates.
  • For large datasets, set max_eps to limit computation; use reachability plot to guide threshold selection.
  • Validate clusters visually using reachability valleys combined with silhouette when threshold is fixed.
