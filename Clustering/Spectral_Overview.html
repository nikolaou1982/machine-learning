Spectral Clustering Overview Guide

Graph-based clustering using eigenvalues and eigenvectors to partition data into clusters in a lower-dimensional embedding space.

Primary Uses

Image segmentation, community detection in networks, non-convex cluster shapes, data with complex geometric structures, recommendation systems, document clustering.

Strengths

Handles non-convex clusters, works well with complex cluster shapes, good for data that forms connected components in similarity space, no assumption of spherical clusters.

Watch-outs

Computationally expensive (requires eigen-decomposition), sensitive to similarity metric and parameters, requires careful tuning of affinity parameters, scaling issues with large datasets.

How It Works Graph-Based

Spectral clustering treats data as a graph where points are nodes and similarities are edges. It uses the Laplacian matrix of this graph to find a lower-dimensional embedding where clusters are more separable.

  • Affinity Matrix: Build similarity/affinity matrix (RBF kernel or k-nearest neighbors)
  • Graph Laplacian: Compute normalized or unnormalized Laplacian from affinity matrix
  • Eigenvalue Decomposition: Find first k eigenvectors of Laplacian (k = number of clusters)
  • Embedding: Use eigenvectors as new feature space (each point → k-dimensional vector)
  • K-means: Apply K-means on embedded space to get final clusters
Tip: Spectral clustering excels when clusters are non-convex or when simple distance-based methods (like K-means) fail. The spectral embedding makes clusters more separable.

Clustering Statistics Interpretation

Clusters
Number of clusters found
Silhouette Score
Cluster quality (-1 to 1, higher = better)
Total Points
Number of data points
Eigenvalue Gap
Gap between kth and (k+1)th eigenvalue (larger = better cluster separation)

The eigenvalue gap indicates how well-separated clusters are in the embedding space. A large gap suggests natural cluster boundaries.

Reading the Graphs 2D

  • Clustered Data: Final cluster assignments; colors represent different clusters.
  • Affinity Matrix: Heatmap showing pairwise similarities; brighter = more similar; reveals data structure.
  • Eigenvalues Plot: Shows eigenvalues in descending order; look for an "elbow" or gap indicating optimal k.
  • Spectral Embedding: 2D visualization of embedded space (using first 2 eigenvectors); clusters should be more separated here.
  • Silhouette Analysis: Bar chart showing silhouette scores per sample; positive = well-clustered, negative = misclassified.

Reading the Graphs 3D

  • Clustered 3D Data: 3D scatter plot with color-coded clusters.
  • Spectral Embedding (3D): First 3 eigenvectors as 3D coordinates; clusters should be separated in this space.
  • Eigenvalues Plot: Same as 2D; helps select optimal number of clusters.
  • Silhouette Analysis: 3D visualization of silhouette scores, possibly as color or size on points.
  • Affinity Graph: 3D visualization of similarity connections between points (optional).

Parameter Guidance Best Practice

  • n_clusters: Use eigenvalue gap method: plot eigenvalues and look for largest gap between consecutive eigenvalues.
  • Affinity: "rbf" (RBF kernel) for general use, "nearest_neighbors" for sparse/local structure; "rbf" works well for most cases.
  • Gamma: (For RBF) Controls similarity decay; smaller = more local, larger = more global. Try default (1/n_features) or tune via grid search.
  • n_neighbors: (For nearest_neighbors) Number of neighbors in k-NN graph; start with 10-15, increase for more global structure.
  • Eigen Solver: "arpack" (default) for small-medium data, "lobpcg" for faster but less accurate results.

Data Preparation Best Practice

  • Standardize features (affinity measures are scale-sensitive; standardization ensures fair distance calculations).
  • Works best when clusters are connected in similarity space (points in same cluster should have high similarity).
  • For very large datasets, consider dimensionality reduction first (PCA) or use approximate methods.
  • Remove outliers that might create spurious connections in the affinity graph.

Common Pitfalls Avoid

  • Wrong affinity parameter → poor similarity matrix → bad clustering (try both "rbf" and "nearest_neighbors").
  • Too many clusters → over-segmentation (use eigenvalue gap to select k).
  • Too few clusters → under-segmentation (check eigenvalue plot for natural number of clusters).
  • Poor gamma (RBF) → either too local (small gamma) or too global (large gamma); tune via cross-validation.
  • Not standardizing → features with larger scales dominate similarity calculations.

Spectral vs K-means Comparison

  • Cluster Shape: Spectral handles non-convex shapes; K-means assumes convex (spherical) clusters.
  • Complexity: Spectral is O(n³) due to eigen-decomposition; K-means is O(n·k·iterations), much faster.
  • Parameters: Spectral requires affinity type and parameters (gamma/n_neighbors); K-means only needs k.
  • Scalability: K-means scales better; Spectral is limited by eigen-decomposition cost.
  • Use Case: Spectral for complex geometries; K-means for simple, well-separated spherical clusters.
