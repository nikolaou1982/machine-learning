DBSCAN Overview Guide

Density-based clustering: find arbitrarily shaped groups, mark sparse points as noise.

Primary Uses

Geospatial hot-spots, anomaly detection, image/point-cloud segmentation, customer segments, denoising.

Strengths

Arbitrary shapes, no cluster count needed, noise handling, robust to outliers.

Watch-outs

Mixed densities, high dimensions (distance less informative) — consider OPTICS/HDBSCAN.

How It Works Short

Core points have ≥ minSamples neighbors within eps. Core points connect into clusters; nearby points are border; others are noise.

  • eps: neighborhood radius (pick via k-distance elbow)
  • minSamples: density threshold (2D≈4–10, 3D≈6–15 typical)
Tip: Standardize features before DBSCAN so distances are comparable.

Reading the Graphs 2D

  • Original Data: raw geometry; density hints; obvious outliers.
  • Clustering Results: assignments vs noise; validate shape/separation.
  • k-distance: sort k-th neighbor distances (k=minSamples); pick eps at the elbow.
  • Core/Border/Noise: density makeup; many border/noise may mean eps too small or varied densities.
  • eps × minSamples Heatmap: color = silhouette, text = clusters; find stable, high-quality regions.

Reading the Graphs 3D

  • Original 3D: spatial structure and overlap; rotate to inspect separation.
  • Clustering 3D: depth reveals cluster layering missed in 2D.
  • k-distance: same elbow method; values are generally larger vs 2D for similar scales.
  • Core/Border/Noise: border often increases in 3D due to surface effects; tune eps accordingly.
  • eps × minSamples Heatmap: use to find robust parameter zones before exploring visuals.

Data Preparation Best Practice

  • Standardize or normalize features; remove duplicates.
  • Subsample for parameter sweeps on very large datasets.
  • Consider OPTICS/HDBSCAN for varying density scenarios.
