DENCLUE Overview Guide

DENsity-based CLUstEring: uses kernel density estimation and hill-climbing to find density attractors and cluster points by convergence paths.

Primary Uses

Smooth density landscapes, gradient-based clustering, attractor-based grouping, kernel density estimation applications, continuous density surfaces.

Strengths

Handles arbitrary shapes; smooth density estimation; gradient-driven convergence; finds density peaks (attractors); works with continuous density functions.

Watch-outs

Bandwidth (sigma) selection critical; convergence paths can be slow; kernel choice affects results; requires standardized data for meaningful distances.

How It Works Short

Estimates density using Gaussian kernels, then performs hill-climbing from each point along the density gradient to find local density maxima (attractors). Points converging to the same attractor form a cluster. Low-density attractors are treated as noise.

  • sigma (bandwidth): kernel bandwidth controlling smoothness (typical: 0.1–1.0; larger = smoother, fewer clusters)
  • xi (density threshold): minimum density for cluster acceptance (typical: 0.001–0.1; lower = more clusters, higher = more noise)
  • hill-climbing: gradient ascent to find density peaks; convergence tolerance affects precision
Tip: Start with sigma ≈ 0.5, then adjust xi to control noise vs. cluster trade-off. Use gradient field plot to understand density flow.

Reading the Graphs 2D

  • Clustered Data with Attractors: points colored by cluster; red stars show density attractors (cluster centers).
  • Density Heatmap: kernel density estimation surface; peaks = high density, valleys = low density. Attractors sit at local maxima.
  • Attractor Convergence Paths: hill-climbing trajectories showing how points flow to attractors; reveals cluster boundaries and density valleys.
  • Density Gradient Field: vector field showing gradient direction and magnitude; arrows point "uphill" toward attractors; helps understand why points converge to specific attractors.
  • Attractor Basins: colored regions showing which attractor each point converges to; basin boundaries reveal cluster separation; regions without attractors become noise.

Reading the Graphs 3D

  • 3D Clustered Data with Attractors: 3D scatter with clusters and attractors (diamond markers); rotate to inspect spatial structure.
  • 3D Density Isosurface: 3D density visualization showing density levels; isosurfaces wrap high-density regions where attractors lie.
  • Attractor Convergence Paths: 3D trajectories showing hill-climbing paths; paths follow density gradients in 3D space.
  • Density Gradient Field: 3D cone plot showing gradient vectors; cones point toward density peaks; magnitude indicates gradient strength.
  • Attractor Basins (2D Projection): basin membership projected onto XY plane; shows basin boundaries in 2D slice (projected at mid-Z).

Parameter Guidance Tuning

Sigma (Bandwidth):

  • Too small: overfitting, many small clusters, sensitive to noise
  • Too large: underfitting, few large clusters, loses detail
  • Rule of thumb: ~10–20% of data range, or use k-distance plot (similar to DBSCAN)

Xi (Density Threshold):

  • Too low: includes noise as clusters, more clusters found
  • Too high: rejects valid clusters as noise, fewer clusters
  • Adjust based on density heatmap: set above noise floor, below peak density

Data Preparation Best Practice

  • Standardization required: distances must be meaningful; standardize/normalize all features to same scale.
  • Remove duplicates; handle missing values before clustering.
  • Use gradient field and convergence paths to validate sigma choice: gradients should point toward attractors.
  • Inspect attractor basins: well-separated basins indicate good clustering; overlapping basins suggest sigma too large.

Common Pitfalls Avoid

  • Not standardizing data: without standardization, distance metrics are meaningless; all points may converge to one attractor or scatter incorrectly.
  • Ignoring gradient field: if gradient arrows don't point toward attractors, sigma may be wrong; use gradient field to diagnose.
  • Convergence paths crossing basins: paths should stay within their basin; crossing indicates sigma too small or numerical issues.
  • Too many noise points: increase sigma or decrease xi; check density heatmap to see if threshold is appropriate.
  • All points in one cluster: sigma too large; decrease sigma to reveal structure, or increase xi if truly one cluster.
