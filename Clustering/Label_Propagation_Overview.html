Label Propagation Overview Guide

Semi-supervised graph-based clustering algorithm that propagates labels through a network based on neighbor similarity.

Primary Uses

Semi-supervised learning, community detection in networks, image segmentation, text classification, recommendation systems, social network analysis, identifying cohesive groups in graphs.

Strengths

Fast and simple algorithm (O(n) per iteration), works well with few labeled examples, no need to specify number of clusters, naturally handles graph structure, good for semi-supervised learning, can work on weighted and unweighted graphs.

Watch-outs

May converge to trivial solutions (single label dominates), sensitive to initial labels, can be unstable for certain graph structures, may not handle disconnected components well, requires graph connectivity for propagation.

How It Works Label Propagation

Label Propagation is an iterative algorithm that spreads labels through a graph by having each node adopt the label that is most frequent among its neighbors. It's particularly useful for semi-supervised learning where you have some labeled data and want to propagate labels to unlabeled data.

  • Initialization: Each node starts with its own unique label (or given labels if semi-supervised).
  • Iteration: For each node, find the most frequent label among its neighbors (weighted by edge weights if available).
  • Update: Update the node's label to the most frequent neighbor label.
  • Convergence: Repeat until labels stabilize (no changes) or maximum iterations reached.
  • Result: Nodes with the same label form communities/clusters.
Tip: Label Propagation works best when the graph has clear community structure. For spatial data, convert to a graph using k-nearest neighbors or distance-based thresholds. The algorithm converges quickly when communities are well-separated.

Clustering Statistics Interpretation

Clusters
Number of unique labels (communities) found
Iterations
Number of iterations until convergence
Total Points
Number of data points (nodes)
Modularity
Quality measure for community structure (higher = better)

Modularity measures how well-separated communities are. Values typically range from 0 to 1, with > 0.3 indicating good community structure. The algorithm automatically determines the number of clusters based on graph structure.

Reading the Graphs 2D

  • Clustered Data: Scatter plot showing final cluster assignments; same color = same community.
  • Graph Visualization: Network view showing nodes (points) and edges (connections); edges connect nearby points based on k-NN or distance threshold.
  • Label Propagation History: Shows how labels change over iterations; convergence when no more changes occur.
  • Modularity Evolution: Tracks modularity score as algorithm progresses; should stabilize at convergence.
  • Cluster Size Distribution: Bar chart showing size of each community; balanced sizes indicate natural communities.

Reading the Graphs 3D

  • Clustered 3D Data: 3D scatter plot with color-coded clusters; rotate to explore structure.
  • Graph Visualization (3D): 3D network view showing spatial relationships and connections; helps visualize community boundaries.
  • Label Propagation History: Same convergence tracking as 2D; monitor algorithm progress.
  • Modularity Evolution: Same quality measure tracking; observe convergence behavior.
  • 3D Community Structure: Spatial separation of communities in 3D space; useful for understanding cluster boundaries.

Parameter Guidance Best Practice

  • k (Neighbors): Number of nearest neighbors for graph construction; start with k=5-15, increase for denser graphs, decrease for sparser graphs.
  • Distance Threshold: Alternative to k-NN; connect points within this distance; useful for controlling graph density.
  • Max Iterations: Maximum iterations before stopping (default usually sufficient); increase if convergence is slow.
  • Graph Type: k-NN graph (most common) or distance-based threshold; k-NN works well for most cases.
  • Weight Function: How to weight edges (uniform, distance-based, RBF); distance-based often works well for spatial data.

Data Preparation Best Practice

  • Standardize features before converting to graph (ensures fair distance calculations).
  • For 2D/3D spatial data, convert to k-NN graph or distance-threshold graph.
  • For pre-existing graphs, use directly (edge list format).
  • Remove outliers that might create spurious connections.
  • Ensure graph is connected (or handle disconnected components separately).

Common Pitfalls Avoid

  • Single dominant label → all nodes get same label (try different k values or distance thresholds).
  • Too many small clusters → graph is too sparse (increase k or decrease distance threshold).
  • Too few clusters → graph is too dense (decrease k or increase distance threshold).
  • Non-convergence → increase max iterations or check for disconnected components.
  • Poor graph construction → k-NN graph quality affects results; tune k carefully.

Label Propagation vs Other Methods Comparison

  • vs Louvain Method: Label Propagation is simpler and faster but less accurate; Louvain uses modularity optimization and produces better results.
  • vs Spectral Clustering: Label Propagation is faster (O(n)) but Spectral is more precise; Spectral requires eigen-decomposition.
  • vs K-means: Label Propagation works on graph structure; K-means works on point coordinates; different use cases.
  • vs Girvan-Newman: Label Propagation is much faster; Girvan-Newman is slower but produces hierarchical structure.
  • vs Semi-supervised Learning: Label Propagation is specifically designed for semi-supervised scenarios with few labeled examples.

Graph Construction from Spatial Data Concept

For 2D/3D point datasets, we convert spatial data to a graph where points are nodes and edges connect similar points. This allows Label Propagation to work on spatial data.

  • k-NN Graph: Each point connects to its k nearest neighbors; creates local connectivity.
  • Distance Threshold: Connect all points within a certain distance; creates density-based connectivity.
  • Edge Weights: Can use uniform weights or distance-based weights (closer = stronger connection).
  • Graph Quality: Good graph construction is crucial; k-NN with k=5-15 typically works well.
Note: The graph construction method (k-NN vs distance threshold) significantly affects results. For most spatial datasets, k-NN graphs with k=5-10 work well. Experiment with different k values to find optimal community structure.
