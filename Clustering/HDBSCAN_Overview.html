HDBSCAN Overview Guide

Hierarchical density-based clustering: robust to variable density, extracts stable clusters with outlier scores.

Primary Uses

Varying-density clustering, anomaly detection with outlier scores, geospatial/event hot-spots, point clouds.

Strengths

Handles variable densities; no single eps; stability-based selection; hierarchy for multi-scale structure.

Watch-outs

Parameter interactions (min_cluster_size, min_samples); interpretation of hierarchy requires care.

How It Works Short

Builds a mutual-reachability MST, condenses it across distance levels, then selects clusters by stability (EOM). Outliers receive scores.

  • min_cluster_size: smallest allowable cluster (stability constraint)
  • min_samples: core-ness; larger → more conservative, better noise handling
Tip: Start with min_cluster_size as ~1–5% of N; increase min_samples for noisy data.

Reading the Graphs 2D

  • Clustering Results: stable cluster assignments vs noise.
  • Hierarchy: merge distance and structure; inspect where clusters split/merge.
  • Membership Probability: confidence per point (proxy), highlights uncertain borders.
  • Stability Bars: relative robustness of clusters (proxy).
  • Condensed Tree: hierarchy condensed; bubble size ~ component size.

Reading the Graphs 3D

  • 3D Clustering: spatial separation; rotate to inspect overlaps.
  • Hierarchy: same as 2D; examine multi-scale structure.
  • Membership Probability: colored 3D scatter (proxy) to spot uncertain regions.
  • Stability Bars: compare cluster robustness in 3D.
  • Condensed Tree: proxy overview of merges vs distance.

Data Preparation Best Practice

  • Standardize/normalize; remove duplicates; consider subsampling for sweeps.
  • Use domain knowledge to validate clusters beyond a single metric.
